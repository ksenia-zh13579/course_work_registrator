# Registrator — Серверная часть

Серверная часть проекта реализована с помощью стэка Node.js + Express + Prisma + PostgreSQL. API доступно по базовому пути `/api` и подключается к фронтенду на `http://localhost:5173`.

## Возможности

- регистрация, вход, обновление токена и выход из системы пользователя
- профили пользователей с возможностью чтения и редактирования
- управление происшествиями: загрузка из базы данных, создание, редактирование и удаление
- управление участниками и их статусами
- получение списков типов и статусов происшествий для форм фронтенда
- OpenAPI-документация через Swagger UI

## Установка

1. Перейдите в папку сервера:

```bash
cd server
```

2. Установите зависимости:

```bash
npm install
```

3. Настройте подключение к базе данных PostgreSQL. Проект использует Prisma. Проверьте переменные окружения в `.env` (если используется) для `DATABASE_URL`.

4. Выполните миграции и загрузите начальные данные:

```bash
npm run migrate
npm run seed
```

## Запуск

- Для разработки с автоматической перезагрузкой при изменениях:

```bash
npm run dev
```

- Для запуска в продакшн-режиме:

```bash
npm start
```

Сервер по умолчанию запускается на порту `3001`.

## Swagger документация

После запуска сервера API документация доступна по адресу:

- `http://localhost:3001/api-docs`
- `http://localhost:3001/api-docs-json`

## Структура проекта

- `src/server.js` — точка входа, настройка Express, CORS, Swagger, маршрутов и ошибок
- `src/routes/` — маршруты API
- `src/controllers/` — бизнес-логика для каждого набора маршрутов
- `src/middlewares/` — middleware для аутентификации, авторизации, валидации и обработки ошибок
- `src/validators/` — Zod-схемы валидации запросов
- `src/docs/` — генерация OpenAPI схемы и регистрация документации
- `prisma/schema.prisma` — модель базы данных
- `prisma/seed.js` — заполнение начальных данных
- `src/tests/` — тесты
    - `src/tests/unit/` — модульные тесты (контроллеры, утилиты, валидаторы)
    - `src/tests/integration/` — интеграционные тесты (API endpoints)

## Базовый путь API

`http://localhost:3001/api`

## Аутентификация и авторизация

- `POST /api/auth/register` — регистрация
- `POST /api/auth/login` — вход, возвращает access token и  добавляет refresh token в cookie
- `POST /api/auth/refresh` — обновление access token с помощью refresh token
- `DELETE /api/auth/logout` — выход из системы, удаление (нужен refresh token)

Для защищенных маршрутов требуется заголовок `Authorization: Bearer <token>`.

## Основные маршруты API

### Пользователь

- `GET /api/profile/` — получить профиль пользователя
- `PATCH /api/profile/` — обновить профиль пользователя

### Происшествия

- `GET /api/incidents/` — получить список происшествий
- `POST /api/incidents/` — создать происшествие
- `PATCH /api/incidents/:id` — обновить происшествие по ID
- `DELETE /api/incidents/:id` — удалить происшествие по ID

### Типы и статусы происшествий

- `GET /api/incidents/form/types` — получить список типов происшествий
- `GET /api/incidents/form/statuses` — получить список статусов происшествий

### Участники

- `GET /api/participants/search` — поиск нужных участников
- `GET /api/participants/` — получить список участников
- `POST /api/participants/` — создать участника
- `PATCH /api/participants/:id` — обновить участника по ID
- `DELETE /api/participants/:id` — удалить участника по ID

### Статусы участников происшествий

- `GET /api/involvements/search` — поиск статусов участников происшествий
- `GET /api/involvements/` — получить список статусов участников происшествий
- `POST /api/involvements/` — добавить статус участника происшествий
- `PATCH /api/involvements/:id` — обновить статус участника происшествий по ID
- `DELETE /api/involvements/:id` — удалить статус участника происшествий по ID

## Технологии

- **Express** 5.2.1 — библиотека для создания и настройки сервера
- **Prisma** 7.8 — ORM для взаимодействия с базой данных
- **Pg** 8.21 — адаптер для настройки работы с PostgreSQL
- **ZOD** 4.4.3 — библиотека для валидации данных http-запросов и ответов 
- **zod-to-openapi** 8.5 — для автоматизации создания документации OpenAPI через Swagger
- **Bcrypt** 6.0 — для хэширования паролей
- **Crypto** 1.0.1 — для генерации UUID
- **Jsonwebtoken** 9.0.3 — для генерации токенов
- **Dotenv** 17.4.2 — для работы с переменным окружения
- **Nodemon** 3.1.14 — для настройки автоматической перезагрузки сервера при изменениях при его запуске в режиме разработки
- **Jest** 30.4.2 — фреймворк для модульного и интеграционного тестирования
- **Supertest** 7.2.2 — библиотека для тестирования HTTP-запросов (интеграционные тесты)
- **Babel** — транспиляция кода для тестов
- **cross-env** — установка переменных окружения для тестов

## Примечания

- Некоторые маршруты публичны для чтения, а записи/изменения требуют аутентификации и соответствующей роли.
- Документация API генерируется автоматически на основе Zod-схем и OpenAPI.
- Для корректной работы CORS-разрешения настроены для фронтенда на `http://localhost:5173`.
