# Registrator — Система регистрации происшествий

Клиент-серверное приложение для регистрации и управления происшествиями с разграничением доступа по ролям пользователей.

## Возможности

- регистрация и аутентификация пользователей с JWT-токенами
- управление профилями пользователей
- создание, редактирование и удаление происшествий
- управление участниками происшествий и их статусами
- фильтрация и поиск данных
- разграничение доступа: гость, пользователь, администратор
- OpenAPI-документация API через Swagger UI
- адаптивный интерфейс для всех устройств

## Быстрый старт

### 1. Клонирование репозитория

```bash
git clone https://github.com/ksenia-zh13579/course_work_registrator.git
cd course_work_registrator
```

### 2. Настройка сервера

```bash
cd server
npm install
```

Настройте подключение к базе данных PostgreSQL в файле .env:
```bash
DATABASE_URL="postgresql://user:password@localhost:5432/registrator"
JWT_SECRET="your-secret-key"
```

Выполните миграции и загрузите начальные данные:
```bash
npm run migrate
npm run seed
```

Запустите сервер:
```bash
npm run dev
```

### 3. Настройка клиента

В новом терминале:
```bash
cd client
npm install
npm run dev
```

Клиент будет доступен по адресу: http://localhost:5173

## Роли пользователей

### Гость

- просмотр главной страницы
- регистрация и вход в систему

### Пользователь

- просмотр и редактирование своего профиля
- просмотр списка происшествий
- создание новых происшествий
- просмотр участников и их статусов

### Администратор

- все права пользователя
- редактирование и удаление происшествий
- управление участниками (добавление, редактирование, удаление)
- управление статусами участников
